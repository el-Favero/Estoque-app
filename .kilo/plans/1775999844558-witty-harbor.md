## Problema Principal

**Cadastromanual item por item é lento e não escalável.**

---

## Solução: Cadastro em Massa

### Opção 1: Importação via planilha/CSV (Recomendado)

Criar tela de importação que:
1. Exporte modelo de planilha (nome, categoria, validade, quantidade, etc)
2. Permita importar arquivo CSV/Excel
3. Parse e valide os dados
4. Mostre预览 com erros destacados
5.-batch cadastre no Firestore

**Prazo estimado:** 2-3 horas

### Opção 2: Pre-cadastro de categorias Itens base

Adicionar lista de produtos comuns do segmento (ex: se for Supermercado, adicionar 50+ itens comuns)

**Prazo estimado:** 1 hora

### Opção 3: Scanner rápido contínuo

Ao escanear item novo, já abre formulário pré-preenchido com dados do Scanner

**Prazo estimado:** 30 minutos

## Decisão Tomada

Com ~100 produtos, a melhor abordagem é:

### 1. Importação via CSV (prioritária)
- Mais flexível: usuário pode criar lista no Excel/Google Sheets
- Formato simples: nome, categoria, validade, quantidade, código de barras
- Validação com erros claros antes do cadastro

### 2. Lista de produtos base
- Adicionar ~30-50 produtos comuns como ponto de partida
- Usuário pode editar/remover después

---

## Exportação (relatórios)

Ambas opções:
- **PDF**: para impressão/compartilhamento
- **Excel/CSV**: para manipulação externa

---

## Plano de Implementação

### Fase 1: Importação CSV
1. Criar tela de importação (app/importar.tsx)
2. Componente de upload (aceitar CSV ou colar texto)
3. Parser com validação
4. Preview dos dados antes de confirmar
5. Batch insert no Firestore

### Fase 2: Lista Base
1. Criar JSON com produtos comunes do segmento
2. Botão na tela de cadastro para "adicionar lista base"
3. Batch insert dos produtos selecionados

### Fase 3: Exportação
1. Adicionar botão "Exportar" na tela de relatórios
2. Gerar PDF usando expo-print
3. Gerar CSV para Excel
